Rijksstudio User Trailing

Evaluating the Rijksstudio's website, part of the Rijksmeum online presence

In this project I evaluated the Rijksstudio website with more 3 colleagues to identify why people use the website and what are the user experience's main issues using the tools. The Rijksstudio is part of the Rijksmuseum website and offers the visitors and users an online experience of the physical museum located in Amsterdam.

Research

Research Plan

The main goal of this User Trailing is to evaluate how effective and efficient the Rijksstudio page of the Rijksmuseum Website is for the designated target group. The main questions we came up to start the research were related to the usability problems might face, how approachable are the tools offered by the website, how a first time user find informations and functions; and the reasons for unclearness for the user.

Research Process

  1. Prepare protocol
  2. User trailing
  3. Data analysis
  4. Recommendations
  5. Conclusion

Protocol

We prepared a protocol were we focused our tasks to the elements we wanted to investigate. We tried the protocol out in a pilot session to get the user experience and check if the protocol is clear and intent for our queres. We tried it out with 4 different international users that had not use it before.

My Results

Issue 1

SEARCH

It is not intuitive where to locate the page to search the artists by their initials.

Issue 2

CREATE

The user was not sure how to create “your own collection” since there are a few options whose meanings are not clear

Issue 3

LANGUAGE

The language suddenly changed from english to dutch

Issue 4

SEARCH

it's not intuitive that the user should search for the initial of the surname, in the "search by artist initials" page.

Issue 5

SEARCH

the order of the artists, when listed by their surname, is not alphabetical.

ACTION
